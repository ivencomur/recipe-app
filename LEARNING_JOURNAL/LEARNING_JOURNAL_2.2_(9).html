What This Exercise Was About

I learned the difference between a Django project (global config, settings, URLs, DB) and an app (modular feature). I created the project structure, ran migrations, started the local server, created a superuser, and briefly explored app creation with a first books app for the Bookstore example.

What I Did (Step-by-Step)

  1. Created workspace & venv: A2_Recipe_App with a2-ve-recipeapp; activated it and installed Django 5.x.
  2. Created Django project: django-admin startproject recipe_project.
  3. Renamed outer folder to src; verified structure with manage.py and inner package.
  4. Ran migrations: python manage.py migrate created db.sqlite3.
  5. Runserver & verify: python manage.py runserver; saw the Django success page.
  6. Created superuser & logged in admin: http://127.0.0.1:8000/admin/ showed my user.
  7. (Optional) Created first app: python manage.py startapp books.

Issues I Hit & Fixes

  • Path typo with ‚Äú...‚Äù caused ‚ÄúNo such file or directory‚Äù. Fix: use a normal relative path like cd src.
  • Windows shells differ for activation. Git Bash: source venv/Scripts/activate; PowerShell: .\venv\Scripts\Activate.ps1.

New Skills Acquired

  • Explained project vs app and walked the generated project files.
  • Ran initial migrations and development server locally.
  • Created a Django superuser and accessed the admin.

What I‚Äôll Remember

manage.py mirrors django-admin but preconfigures project settings. Projects own global config; apps add reusable functionality.
