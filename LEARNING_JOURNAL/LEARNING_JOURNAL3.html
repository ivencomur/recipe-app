4) Problems I Ran Into & How I Fixed Them

My Loop Wasn't Running (Indentation Error)

This was the biggest problem I faced. After writing all my code, I ran the program and it asked "How many recipes?" but then just stopped. It didn't ask me to enter any recipe details at all!
What went wrong: On one line, I had 5 spaces instead of 4. In Python, you have to be super consistent with your spacing (indentation). Even one extra space can confuse Python about which code belongs inside which section.
How I fixed it: I looked carefully at every line and made sure all my indents used exactly 4 spaces. Once I fixed that one line, everything worked! This taught me to be very careful about spacing in Python.

Understanding Nested Loops

At first, having a loop inside another loop was confusing. I had trouble keeping track of which loop was doing what.
How I understood it: I thought about it like this: the outer loop says "collect recipe #1, #2, #3..." and for each recipe, the inner loop says "check ingredient 1, ingredient 2, ingredient 3..." The outer loop runs 3 times total, but the inner loop runs completely each time the outer loop runs once.

Figuring Out the Difficulty Logic

I had to translate the difficulty rules into if/elif/else statements, and there were 4 different possibilities to consider.
How I worked it out: I wrote down all the combinations: quick + simple = Easy; quick + complex = Medium; slow + simple = Intermediate; slow + complex = Hard. Then I turned each one into an if statement. The last one (Hard) didn't need a condition because if none of the other three are true, it must be Hard.

When to Use "not in" vs Other Comparisons

I needed to check if an ingredient was already in my list before adding it, but I wasn't sure about the right way to do this.
What I learned: Python has a special word "in" that checks if something exists in a list. If I write if ingredient not in list, Python looks through the whole list and tells me True or False. This is much easier than writing a loop to check every item myself!

Return vs. Print in Functions

I wasn't sure when to use return and when to use print() inside my function.
What I learned: print() just shows something on the screen. return actually sends data back so other parts of my program can use it. My take_recipe() function uses return because I need to get the recipe dictionary back so I can store it in my list.
